# AudioPM Local 🎙️

> **Часть проекта TaskWave** — AI-платформа, которая превращает записи встреч в структурированную проектную документацию.
>
> Этот репозиторий содержит **локальный CLI-движок** — ядро обработки, которое будет питать бэкенд TaskWave.

---

## Что делает

AudioPM Local берёт запись митинга и автоматически создаёт готовый проектный документ с задачами, эпиками и критериями приёмки — **100% офлайн, 100% приватно, 100% бесплатно**.

```
[Аудиофайл]  →  [Whisper STT]  →  [ИИ Корректор]  →  [LLM Аналитик]  →  [.txt документ]
 MP3/WAV/         Локальная        Исправляет          Генерирует        Эпики, задачи,
 M4A/OGG/...    транскрипция      ошибки жаргона      документацию        DoD, AC
```

---

## Возможности

- 🎙 **Локальное распознавание речи** через [Whisper.net](https://github.com/sandrohanea/whisper.net) — никаких облаков и API-ключей
- 🤖 **Двухшаговый AI-конвейер** на базе [Ollama](https://ollama.ai/):
  - **Шаг А — Корректор:** исправляет ошибки транскрипции (`«делик»` → `Daily`, `«жира»` → `Jira`)
  - **Шаг Б — Аналитик:** генерирует структурированную PM-документацию
- 📋 **Два режима вывода:** иерархия эпиков или плоский список задач
- ✅ **DoD и Acceptance Criteria** — опционально для каждой задачи
- 🔊 **Любой аудиоформат** — MP3, M4A, OGG, FLAC, WAV (автоконвертация через NAudio)
- 🧠 **Три уровня модели Whisper** — Base / Small / Medium, выбор прямо в приложении
- 🖥 **Интерактивный CLI** — чистый TUI-интерфейс на [Spectre.Console](https://spectreconsole.net/)

## Требования

| Зависимость | Версия | Примечание |
|---|---|---|
| [.NET SDK](https://dotnet.microsoft.com/download) | 8.0+ | Для сборки и запуска |
| [Ollama](https://ollama.ai/) | Последняя | Должна быть запущена на `localhost:11434` |
| Модель Ollama | Любая | Например `llama3.2`, `mistral`, `deepseek` |
| Whisper модель | Скачивается автоматически | Base (~145 МБ), Small (~480 МБ), Medium (~1.5 ГБ) |

### Установка и запуск Ollama

```bash
# Скачайте с https://ollama.ai/
# Загрузите любую модель:
ollama pull llama3.2

# Ollama запускается автоматически на Windows/macOS
# На Linux:
ollama serve
```

## Установка

```bash
git clone https://github.com/Piryutko/TaskWave-AI.git
cd audiopm-local/AudioPM.Local
dotnet run
```

При первом запуске приложение:
1. Ищет установленные Whisper-модели на диске
2. Предлагает выбрать модель (Base / Small / Medium)
3. При необходимости автоматически скачивает модель с Hugging Face
4. Показывает доступные модели Ollama для выбора
5. Открывает главное меню

---

## Модели Whisper

| Модель | Размер | Лучше всего для |
|---|---|---|
| **Base** | ~145 МБ | Быстрые тесты, слабые машины |
| **Small** ⭐ | ~480 МБ | Ежедневная работа — лучший баланс скорости и точности |
| **Medium** | ~1.5 ГБ | Технические митинги, профессиональный сленг, максимальная точность |

Модели скачиваются автоматически с [Hugging Face](https://huggingface.co/ggerganov/whisper.cpp) с прогресс-баром.

## Использование

Главное меню работает в цикле — после каждой обработки возвращаетесь сюда:

```
🎤  Анализировать встречу         — обработать новый аудиофайл
🔄  Сменить Ollama модель          — переключить LLM без перезапуска
🔄  Сменить Whisper модель         — переключить STT на лету
❌  Выйти
```

### Процесс обработки

1. **Выбор аудиофайла** — укажите путь к записи
2. **Выбор режима:**
   - *Эпики и декомпозиция* — иерархия: Эпик → Задача → Подзадача
   - *Простой список задач* — плоский нумерованный список
3. **DoD/AC** — включать ли критерии готовности и приёмки
4. **Обработка** — 4 шага с живым прогресс-баром:
   - Инициализация Whisper
   - Локальная расшифровка аудио
   - ИИ исправляет технические ошибки
   - Генерация документации (счётчик токенов в реальном времени)
5. **Результат** — `.txt` файл сохраняется рядом с аудиофайлом

### Формат выходного файла

```
AudioPM_{имя_файла}_{дата_время}.txt
```

В шапке файла — метаданные (дата, использованные модели, время обработки), далее — сгенерированный документ.

## Архитектурные решения

- **Конвертация аудио:** любой формат конвертируется в 16 кГц / 16-bit / mono WAV через `NAudio.MediaFoundationReader` перед передачей в Whisper
- **Стриминг генерации:** Ollama вызывается с `"stream": true` + `HttpCompletionOption.ResponseHeadersRead` — нет таймаутов при длинных ответах
- **Двухпроходный LLM:** Корректор использует минимальный системный промпт (роль редактора), Аналитик — полный PM-промпт — каждая модель сфокусирована на своей задаче
- **Безопасная загрузка:** модели Whisper скачиваются во временный `.tmp` файл и переименовываются по завершении — никаких повреждённых файлов
